# 图(Graph)

## 「直觉版」的图
    图 = 点 + 点之间的关系
    - 点(Vertex/Node): 对象
    - 边(Edge): 关系
   
    现实中的例子:
    - 城市 ↔ 城市之间的道路
    - 员工 ↔ 上司(力扣 3562 题)
    - 网页 ↔ 超链接
    - 课程 ↔ 先修关系
   
    👉 凡是 `关系` 不是线性的, 基本都能抽象成图
  ---

## 图和数据结构的关系
   | 结构    | 本质              |
   | -----  |-----------------|
   | 数组    | 特殊图(线性链)        |
   | 链表    | 特殊图(每点最多 1 条出边) |
   | 树     | 无环 + 连通 的图      |
   | 二叉树   | 限制更强的树          |
   | **图** | 最一般的关系结构        |
   👉 树 = 特殊的图

   👉 所以 DFS / BFS,其实本来就是「图算法」

   ---

## 图的最基本分类(一定要吃透)
### 1️⃣ 有向图 vs 无向图
  **无向图**
  ```
  A —— B
  ```
  - A 能到 B, B 也能到 A
  - 例: 朋友圈、无向道路

  **有向图**
  ```
  A → B
  ```
  - 只能 A 到 B
  - 例: 上下级关系、依赖关系、任务流

  👉 典型: 有向图

  --- 

### 2️⃣ 是否有环
    - 有环图: 能绕一圈回到自己
    - 无环图: 不能
    
   👉 特殊概念: 
   > DAG(有向无环图)

   --- 

### 3️⃣ 连通性(非常重要)
   **无向图**
   - 连通: 任意两点可达
   - 非连通: 多个 `块`

   **有向图**
   - 强连通
   - 弱连通
   --- 

## 图的表示方式(算法实现核心)
   这一步是 **从“概念”到“代码”的桥**

### 1️⃣ 邻接表(最重要)
  ```
  力扣 3562 题:
  0: [1, 2]
  1: [3],
  2: [],
  3: [] 
  ```
   - 空间省
   - 最常用
   - 几乎所有 LeetCode / Rust 实现首选

   👉 以后 80% 的图题都用这个

### 2️⃣ 邻接矩阵
   ```text
       0 1 2
    0 [0 1 1]
    1 [0 0 1]
    2 [0 0 0]
   ```
   - 查询快
   - 空间 `O(n²)`, 大图不适合
   --- 

## 两大神器
   1️⃣ DFS(深度优先)

   本质:
   - 一条路走到黑
   - 递归 / 栈

   用途:
   - 判断是否有环
   - 拓扑排序
   - 连通分量
   - 树 / 图遍历
   ---

   2️⃣ BFS(广度优先)
   - 一层一层扩展
   - 队列
   
   用途:
   - 最短路径(无权图)
   - 层级关系
   - 拓扑排序(Kahn)
   ---

   📌 重点提醒
   > BFS / DFS 一定要记住:
   >
   > **在图中必须有 visited，否则会死循环**
   ---

## 学习路线
1. 第一阶段: 基础
   - 图的表示(邻接表)
   - DFS / BFS 遍历
   - 连通性问题
   - 判断是否有环

2. 第二阶段: 有向图核心
   - 拓扑排序
   - DAG 动态规划
   - 入度 / 出度思想

3. 第三阶段:进阶
   - 最短路径(BFS / Dijkstra)
   - 并查集
   - 强连通分量(Tarjan / Kosaraju)
